# How `build_state_datasets.py` Gets Its Data

The script does **not** use the Census API. It pulls from three live sources plus one local Census file.

| Output | Source | How the script gets it |
|--------|--------|-------------------------|
| **Real GDP per capita** | BEA (via **FRED**) | HTTP GET to `fred.stlouisfed.org/graph/fredgraph.csv?id={ST}RQGSP` (e.g. `CARQGSP`) |
| **Personal income per capita** | BEA (via **FRED**) | Same FRED CSV endpoint for `{ST}OTOT` (total personal income) |
| **Population** (for per-capita) | **BEA** directly | Downloads `SQINC.zip` from `apps.bea.gov`, reads `SQINC1__ALL_AREAS_1948_2025.csv` |
| **Unemployment** (monthly + quarterly) | **BLS** | POST to `api.bls.gov/publicAPI/v2/timeseries/data/` with LAUS series IDs like `LASST060000000000003` for CA. Optional `BLS_API_KEY` env var for higher rate limits |
| **Homeownership** | **Census HVS** | **Local file only** — parses `tab3_state05_2026_hmr.xlsx` on disk (Table 3 from the Housing Vacancy Survey). No API call |

## Data flow

```
FRED (BEA GDP + income)  ──┐
BEA zip (population)     ──┼──► merge / compute per capita ──► CSVs in output/
BLS API (unemployment)   ──┤
Local Census xlsx (home) ──┘
```

## Census connection

Homeownership data is Census *content*, but you downloaded the Excel export yourself. The script reads that file with `openpyxl` via `filter_homeownership_to_tidy.py` — equivalent to opening the spreadsheet manually, not calling `api.census.gov`.

## Offline alternative

The `SQGDP (1)/` folder is a BEA bulk export that could replace the FRED GDP pulls offline; the current script does not use it yet.

## `build_state_datasets_v1.py`

The v1 script is simpler: **FRED only** for GDP, income, and unemployment — no BLS API, no BEA zip, and no homeownership step.

## Output files

Written to `GDP/output/`:

- `real_gdp_per_capita_by_state.csv`
- `personal_income_per_capita_by_state.csv`
- `monthly_unemployment_rate_by_state.csv`
- `unemployment_rate_by_state.csv`
- `homeownership_rate_by_state.csv`
- `README_state_data.md` (generated by the script)
